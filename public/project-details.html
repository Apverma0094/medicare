Medicine Store — Project Details

Chhota sa intro

Yeh file aapke Laravel project ka structure, request lifecycle, aur commonly used dev commands Hindi mein samjhati hai. Aapke workspace ke files aur folders ke references bhi diye gaye hain.

1) Top-level files / folders aur unka kaam

  • artisan — Laravel CLI executable (php artisan ...).
  • composer.json — PHP dependencies & autoload config.
  • package.json, vite.config.js — frontend build (Vite).
  • public/ — webroot; entry point: public/index.php.
  • bootstrap/ — bootstrapping files; bootstrap/app.php creates the app instance.
  • app/ — application core (Controllers, Models, Providers).
  • config/ — configuration files.
  • database/ — migrations, factories, seeders.
  • resources/ — Blade views and frontend source (css/js).
  • routes/ — route definitions (web.php, console.php).
  • storage/ — logs, compiled views, uploaded files (writable hona chahiye).
  • vendor/ — Composer dependencies (framework, libraries).
  • tests/ — PHPUnit tests (Feature, Unit).

2) Request lifecycle — step-by-step

  1. Browser → Server → public/index.php (autoload + bootstrap/app.php).
  2. bootstrap/app.php — Laravel application instance banata hai.
  3. HTTP Kernel (app/Http/Kernel.php) middleware stack ko chalata hai.
  4. Routes (routes/web.php) match hote hain; controller action call hota hai.
  5. Controller (app/Http/Controllers/*) business logic: validate, call models/services, return view/JSON.
  6. Models (app/Models/*) Eloquent ke through DB queries karte hain.
  7. Views (resources/views/*.blade.php) Blade templates compile hote hain aur response return hota hai.
  8. Response → Kernel → public/index.php → Browser.

5) Contract (input/output) aur edge-cases

Contract (controller action):

  • Input: HTTP Request (params, body, auth)
  • Output: Blade view with data OR JSON
  • Error modes: validation errors (422), auth (401/403), DB errors (500)

Edge cases: missing .env, wrong DB creds, storage permissions, CSRF on forms, large queries (use pagination).
